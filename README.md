# PanelFlow

**Smart comic & manga converter for small screens.** Converts PDF / CBZ / CBR / ZIP / images into e-reader-ready files (CBZ, PDF, fixed-layout EPUB, image ZIP, or the PanelFlow `.pfc` container), with a computer-vision **smart mode** that detects panels, trims margins, splits double pages and re-paginates webtoons for Kindle, Kobo, Boox, PocketBook, Xteink X4 and phones.

- **Stateless & private**: no database, no accounts, no file storage. Everything is processed in memory and discarded when the request ends.
- **Two modes**: *Normal* (pure format conversion, lossless repack when possible) and *Smart* (panel-aware optimization per device profile).
- **Vercel-native**: Next.js frontend + a Python (FastAPI) serverless function.

## Architecture

Bilingual: English at `/` (plus `/guide`, `/privacy`) and Spanish at `/es` (`/es/guia`, `/es/privacidad`), with hreflang alternates, `sitemap.xml` and `robots.txt` generated by Next.

```
src/                 Next.js 15 + React 19 + Tailwind 4 frontend
api/index.py         FastAPI app -> deployed by Vercel as a Python function
api/_core/           Conversion engine (pure, stateless, in-memory)
  readers.py         PDF / CBZ / CBR / ZIP / images -> lazy page stream
  smart.py           OpenCV panel detection, margins, spreads, webtoon mode
  pipeline.py        Orchestration + lossless repack fast path
  writers.py         CBZ / PDF / EPUB (fixed layout) / images ZIP / .pfc
  profiles.py        Device profiles (resolution, grayscale, quality)
```

`/api/py/*` requests are rewritten to the FastAPI app (see `next.config.mjs`); in development they proxy to a local uvicorn server.

## Local development

Requires Node 20+ and Python 3.11+.

```bash
npm install
pip install -r requirements.txt

# terminal 1 — API
npm run dev:api        # uvicorn on :8000

# terminal 2 — web
npm run dev            # next dev on :3000
```

Open http://localhost:3000.

## Deploying to Vercel

1. Push the repo to GitHub and import it in Vercel — Next.js and `api/index.py` are detected automatically; `vercel.json` raises the function's memory/duration for heavy conversions.
2. (Optional) Set the AdSense env vars from `.env.example` in the Vercel dashboard. Ads render only when configured.
3. Notes on platform limits:
   - Direct uploads through a serverless function are limited by Vercel's request-body size (~4.5 MB on some plans). For large comics (100–200 MB CBZ), enable **Vercel Blob client uploads**: the browser uploads to a temporary blob, the function downloads/converts/deletes it. The pipeline already works on plain bytes, so this only changes the transport layer.
   - `maxDuration: 300` in `vercel.json` requires Fluid Compute (default on current Vercel plans).
   - CBR extraction tries three backends in order: the `unrar-cffi` library (if installed), `unrar`/WinRAR (auto-detected on Windows), and `bsdtar`/libarchive (ships with Windows 10/11). Locally at least one is almost always available. On Vercel none may exist — but most `.cbr` files in the wild are actually ZIPs, which are detected by magic bytes and handled regardless; genuine RARs get a clear error suggesting CBZ.

## Privacy model

The engine never touches disk: inputs arrive as `bytes`, pages decode lazily one at a time, and outputs leave as `bytes`. There is no persistence layer to leak into — see `src/app/privacy/page.tsx` for the user-facing policy.

## Monetization

Google AdSense only, deliberately minimal: one banner on the home page, one on the download step, both height-reserved (no layout shift), no popups/interstitials/auto-play — compliant with AdSense best practices.
